## 一、"立即沟通"按钮点击后无效

### 1.1 现象

在以下5个页面点击「立即沟通」按钮：

- 首页推荐岗位 (`index.vue`)
- 职位搜索页 (`Jobs.vue`)
- 公司详情页 (`CompanyDetail.vue`)
- 收藏岗位页 (`FavoriteJobs.vue`)
- HR人才搜索页 (`TalentSearch.vue`)

点击后要么没反应，要么弹出了聊天窗口但看不到会话列表。

### 1.2 根因

问题出在两层：

**第一层：会话列表没有提前加载**

每个页面的 `handleChat` 方法逻辑是：
1. 调用 `createConversation(hrUserId)` 创建一个新会话，拿到 `conversationId`
2. 把 `conversationId` 传给 `ChatPanel` 组件
3. `ChatPanel` 收到后尝试在 `contactList` 里找这个会话

但 `ChatPanel` 的 `contactList` 是在 `onMounted` 里调的——而 `ChatPanel` 组件可能早在页面初始化时就挂载过了，此时 `contactList` 里只有旧数据。新创建的会话 ID 不在旧列表里，所以找不到，UI 上表现为空白。

**第二层：缓存逻辑在每个页面里重复实现**

5 个页面各自写了 20~30 行几乎一样的代码：
- 从 `chatCache` 读缓存
- 调 `createConversation` 创建会话
- 调 `setConversations` 写缓存
- 调 `getCachedConversations` 刷新

这些代码只是复制粘贴，一旦逻辑需要调整，5 个地方都要改。

### 1.3 修复

把缓存和会话选择逻辑**全部收归到 `ChatPanel` 组件内部**，对外只暴露一个 `targetConversationId` prop。

**修复后的 `ChatPanel.vue`** 新增一个 watcher：

```javascript
// 监听外部传入的 targetConversationId
watch(() => props.targetConversationId, async (newId) => {
  if (!newId) return
  // 已经在当前会话，不重复操作
  if (currentContact.value?.id === newId) return

  // 先在已有列表中找
  const existing = contactList.value.find(c => c.id === newId)
  if (existing) { selectContact(existing); return }

  // 找不到就从服务端刷新列表再找
  await refreshAll()
  const found = contactList.value.find(c => c.id === newId)
  if (found) selectContact(found)
})
```

**修复后每个页面的 `handleChat`** 从 ~30 行简化到 ~10 行：

```javascript
const handleChat = async () => {
  if (!detailJob.value?.hrUserId) {
    ElMessage.warning("该职位暂无HR信息")
    return
  }
  try {
    const res = await createConversation(detailJob.value.hrUserId)
    if (res.code === 1 && res.data) {
      targetConversationId.value = res.data.id
      chatVisible.value = true   // 打开弹窗，剩下的交给 ChatPanel 的 watcher
    } else {
      ElMessage.error(res.msg || "创建会话失败")
    }
  } catch {
    ElMessage.error("网络异常，无法创建会话")
  }
}
```

**核心思路**：页面只负责「创建会话」这一件事，拿到 ID 后传给 `ChatPanel`，组件自己负责查列表、刷新、选中。页面不需要知道缓存的存在。

### 1.4 涉及文件

| 文件 | 改动 |
|---|---|
| `src/components/ChatPanel.vue` | 新增 watcher，新增 `refreshAll()` |
| `src/views/user/index.vue` | 精简 `handleChat` |
| `src/views/user/Jobs.vue` | 精简 `handleChat` |
| `src/views/user/CompanyDetail.vue` | 精简 `handleChat` |
| `src/views/user/profile/FavoriteJobs.vue` | 精简 `handleChat` |

## 二、登录后 loading 一直转圈，页面不跳转

### 2.1 现象

输入账号密码点击登录，请求成功返回了 token，但：
- loading 动画一直转，不停
- 页面不跳转到目标页
- 必须手动刷新浏览器才能进入系统

### 2.2 排查过程

登录代码原本是：

```javascript
if (res.code === 1) {
  localStorage.setItem("token", res.data.token)
  localStorage.setItem("loginUser", JSON.stringify(res.data))
  ElMessage.success("登录成功")
  router.replace(target)   // 不 await，直接跳
}
```

之前用户反馈「登录后没反应」，于是有人在 `router.replace(target)` 前面加了 `await`：

```javascript
if (res.code === 1) {
  // ...
  ElMessage.success("登录成功")
  await router.replace(target)   // 加了 await
}
```

加了 `await` 之后，反而 loading 转个没完。

### 2.3 根因

`router.replace(target)` 返回的是一个 Promise。这个 Promise 在以下情况**永远不会 resolve**：

1. `router.replace('/hr/dashboard')` 开始导航
2. 目标页面的 `onMounted` 里调了 API（比如 `checkCompanyStatus()`）
3. 这个 API 返回 401（因为 token 还没生效或时序问题）
4. 拦截器检测到 401，执行 `router.replace('/login')`
5. 这导致第 1 步的导航被**取消**
6. 被取消的导航 Promise 永远不会 resolve

于是 `await router.replace(target)` 就永远卡住，后面的代码（包括 `finally` 里的 `loading.value = false`）永远执行不到，loading 转圈停不下来。

**简单说**：加了 `await` 之后，代码在等一个永远不会完成的 Promise。

### 2.4 修复

去掉 `await`，在跳转前**先关掉 loading**，然后用 `return` 直接退出：

```javascript
if (res.code === 1) {
  localStorage.setItem("token", res.data.token)
  localStorage.setItem("loginUser", JSON.stringify(res.data))
  resetHrStatus()
  ElMessage.success("登录成功")

  loading.value = false          // 先关 loading
  router.replace(target)          // 不 await，触发导航即可
  return                          // 直接返回，跳过 finally
}
```

**关键点**：
- `router.replace()` 不 await——我们不需要等导航完成，只需要「发起导航」
- `loading.value = false` 放在导航**之前**——即使导航被取消，loading 也已经关了
- 用 `return` 跳过 try 块后续代码——不需要再走 `finally`

### 2.5 涉及文件

| 文件 | 改动 |
|---|---|
| `src/views/login/index.vue` | 移除 `await`，调整 loading 关闭时机 |

## 三、聊天面板联系人超过 4 个时无法滚动

### 3.1 现象

当用户和超过 4 个 HR 有过对话后，左侧联系人列表只显示前 4 个，下面的联系人看不到，也没有滚动条。

### 3.2 根因

`ChatPanel.vue` 的布局使用 flexbox，联系人列表区域没有设置 `overflow-y: auto` 和 `min-height: 0`。

在 flex 布局中，子元素默认 `min-height: auto`，意味着内容有多高它就撑多高，不会触发滚动。同时缺少 `overflow-y: auto` 导致超出部分直接被裁剪。

### 3.3 修复

给联系人列表外层包一个 div，设置 flex 溢出滚动：

```html
<!-- 之前 -->
<div class="contact-list">
  <!-- 联系人卡片 -->
</div>

<!-- 之后 -->
<div class="contact-scroll">          <!-- 新增滚动容器 -->
  <div class="contact-list">
    <!-- 联系人卡片 -->
  </div>
</div>
```

```css
.contact-scroll {
  flex: 1;              /* 占据剩余空间 */
  overflow-y: auto;     /* 内容超出时显示滚动条 */
  min-height: 0;        /* flex 子元素允许收缩 */
}

.list-header {
  flex-shrink: 0;       /* 头部标题不收缩 */
}
```

**核心思路**：在 flex 容器中，只有设置了 `min-height: 0` + `overflow-y: auto` 的子元素才会在内容溢出时出现滚动条。

### 3.4 涉及文件

| 文件 | 改动 |
|---|---|
| `src/components/ChatPanel.vue` | 模板加 `.contact-scroll` 容器，CSS 加 flex 溢出样式 |

## 四、前端代码冗余清理

### 4.1 问题概述

逐模块审查后发现以下重复/冗余问题：

| 问题 | 重复次数 | 涉及文件 |
|---|---|---|
| `parseTags` 函数局部定义 | 5 处 | index.vue, Jobs.vue, CompanyDetail.vue, FavoriteJobs.vue, JobApplications.vue |
| `fmtSalaryShort` 函数局部定义 | 2 处 | TalentSearch.vue, TalentPool.vue |
| `maxEducationLabel` 和 `educationLabel` 功能完全一样 | 2 个函数 | MyResume.vue, AiMatch.vue, InterviewManage.vue |
| `parseSkillsArr` 包装函数（调了又包一层） | 3 处 | Dashboard.vue, ApplicationManage.vue, AiMatch.vue |
| 举报弹窗逻辑重复 | 4 处 | Company.vue, Jobs.vue, CompanyDetail.vue, FavoriteJobs.vue |

### 4.2 根因

开发过程中每个页面独立编写，没有及时抽取公共逻辑。属于典型的「复制粘贴式开发」积累的技术债。

### 4.3 修复

**`parseTags`** —— 在 `src/utils/format.js` 中统一定义：

```javascript
export function parseTags(tags) {
  if (!tags) return []
  if (Array.isArray(tags)) return tags
  if (typeof tags === 'string') {
    try { return JSON.parse(tags) } catch { return tags.split(',') }
  }
  return []
}
```

所有页面删除局部定义，改为 `import { parseTags } from "@/utils/format"`。

**`educationLabel` / `maxEducationLabel`** —— 合并为一个函数：

两个函数的映射表完全一样（`'不限' → 1, '高中' → 2, ...`），只是函数名不同。删除 `maxEducationLabel`，全部改用 `educationLabel`。

**`parseSkillsArr`** —— 删除包装层：

```javascript
// 之前：多余包装
const parseSkillsArr = (skills) => parseSkills(skills)

// 之后：直接用
parseSkills(item.skills)
```

**举报弹窗** —— 提取 `ReportDialog.vue` 共享组件：

```javascript
// ReportDialog.vue 的 props
const props = defineProps({
  visible: Boolean,
  targetType: Number,      // 1=职位 2=公司
  targetId: [Number, String],
  targetName: String,
  label: String,           // 显示标签："职位" / "公司"
})
```

4 个页面统一替换为 `<ReportDialog v-model:visible="..." :target-type="..." ... />`。

## 总结：今天修了哪些东西

| 问题 | 严重程度 | 根因一句话 | 修复方式 |
|---|---|---|---|
| 立即沟通无效 | 高 | ChatPanel 收到新 ID 不会主动刷新列表 | ChatPanel 内部加 watcher 监听 prop 变化 |
| 登录转圈卡死 | 高 | `await router.replace` 在导航被取消时永不 resolve | 去掉 await，先关 loading 再跳转 |
| 联系人列表无滚动 | 中 | flex 子元素缺少 `overflow-y: auto` 和 `min-height: 0` | 加滚动容器 + CSS |
| 代码冗余 | 中 | 多个页面复制粘贴相同逻辑 | 提取共享函数和组件 |
| 公司信息排版 | 低 | 公司名不可点击，布局拥挤 | flex 布局 + 链接样式 |
| 后端缺 companyName | 中 | VO 没有公司名字段 | 批量查询填充 |
