# 系统介绍

智能就业服务平台是一套基于 Spring Boot + Vue 3 的全栈招聘管理系统，面向求职者、HR 和管理员提供完整的招聘业务流程。

## 解决的问题

### 1. 求职者找工作效率低

传统招聘平台信息杂乱，求职者难以快速匹配合适岗位。本平台通过 **AI 技能匹配算法** 自动分析简历技能标签与岗位要求的重合度，按匹配分数排序推荐，提高投递精准度。

### 2. HR 招聘流程分散

HR 通常在多个系统间切换：发布职位、筛选简历、安排面试、沟通候选人。本平台提供 **一站式招聘流程管理**，从职位发布到录用全程在线完成，支持状态流转和实时消息沟通。

### 3. 面试管理混乱

传统面试邀约依赖电话或邮件，状态跟踪困难。本平台实现 **面试全生命周期管理**：邀约发起 → 求职者确认 → 面试结果反馈 → 自动流转到录用，状态实时可查。

### 4. 管理员权限粒度不足

多数系统管理员权限粗放，无法区分封禁与删除等敏感操作。本平台实现 **分级权限模型**：普通管理员可封禁/解封账号，仅超级管理员（ID=1）可删除账号和创建管理员账户。

### 5. 数据可视化和决策支持缺失

本平台提供 **实时数据大屏**（管理端概览）和 **HR 数据分析**（投递趋势图、面试转化漏斗），支持按时间范围筛选，帮助管理者掌握招聘全貌。

### 6. 实时沟通不便

通过 **WebSocket 即时通讯**，求职者与 HR 可直接在平台内聊天，无需跳转到第三方工具，消息实时送达并支持未读提醒。

## 系统功能模块

### 求职者端

| 模块 | 功能 |
|------|------|
| 首页 | 推荐岗位（AI 匹配排序）、热门岗位、平台统计数据 |

## 数据库设计概要

数据库共 **18 张表**，覆盖 9 大业务模块：

| 模块 | 表 | 说明 |
|------|-----|------|
| 用户与权限 | users | 统一用户中心，多角色（求职者/HR/管理员） |
| 企业与职位 | companies, jobs | 企业信息与招聘岗位 |
| 简历 | resumes, resume_educations, resume_experiences, resume_projects | 主表 + 三张子表结构 |
| 职位投递 | applications | 投递记录，状态流转核心 |
| 面试邀约 | interview | 面试全生命周期管理 |
| 即时通讯 | conversations, messages | WebSocket 实时聊天 |
| AI 匹配 | match_records, match_skill_details | 技能匹配评分与明细 |
| 收藏与人才库 | job_favorites, company_resume | 双向收藏 |
| 系统反馈与日志 | feedbacks, operation_logs | 工单与审计日志 |

详细设计参见"数据库设计"标签页。
