# 数据库设计文档（招聘系统）

## 一、数据库概述

本数据库用于招聘系统业务开发，主要包含以下核心模块：

- 用户与权限模块
- 企业与职位模块
- 简历模块
- 职位投递模块
- 面试邀约模块
- 即时通讯模块
- AI智能匹配模块
- 收藏与人才库模块
- 系统反馈与日志模块

数据库名称：

```sql
recruitment_db
```

数据库采用：

- MySQL 8.0
- UTF8MB4字符集
- InnoDB存储引擎

设计原则：

- 尽量采用逻辑外键，减少数据库级联压力
- 高频查询字段建立索引
- 适配Redis缓存与AI推荐业务
- 支持高并发招聘场景

# 三、核心业务关系

## 用户与企业关系

```text
users(1) ---> companies(N)
```

## 企业与职位关系

```text
companies(1) ---> jobs(N)
```

## 用户与简历关系

```text
users(1) ---> resumes(N)
```

## 职位投递关系

```text
users ---> applications ---> jobs
```

## 面试邀约关系

```text
applications(1) ---> interview(N)
```

## 聊天关系

```text
conversations(1) ---> messages(N)
```

## AI匹配关系

```text
jobs ---> match_records <--- resumes
```

# 三、核心业务关系

## 用户与企业关系

```text
users(1)  --->  companies(N)
```

一个HR用户可关联一个企业。

------

## 企业与职位关系

```text
companies(1) ---> jobs(N)
```

一个企业可以发布多个职位。

------

## 用户与简历关系

```text
users(1) ---> resumes(N)
```

一个求职者可创建多份简历。

------

## 职位投递关系

```text
users ---> applications ---> jobs
```

求职者通过简历投递职位。

------

## 面试邀约关系

```text
applications(1) ---> interview(N)
```

一次投递可关联多轮面试。

------

## 聊天关系

```text
conversations(1) ---> messages(N)
```

一个会话包含多条消息。

------

## AI匹配关系

```text
jobs ---> match_records <--- resumes
```

AI对岗位与简历进行智能评分匹配。
