# 数据库设计文档（招聘系统）

## 一、数据库概述

本数据库用于招聘系统业务开发，主要包含以下核心模块：

- 用户与权限模块
- 企业与职位模块
- 简历模块
- 职位投递模块
- 面试邀约模块
- 即时通讯模块
- AI智能匹配模块
- 收藏与人才库模块
- 系统反馈与日志模块

数据库名称：

```sql
recruitment_db
```

数据库采用：

- MySQL 8.0
- UTF8MB4字符集
- InnoDB存储引擎

设计原则：

- 尽量采用逻辑外键，减少数据库级联压力
- 高频查询字段建立索引
- 适配Redis缓存与AI推荐业务
- 支持高并发招聘场景

| ip_address    | varchar(45)  | IP地址       | 支持IPv6 |
| remark        | varchar(255) | 操作备注     |          |
| created_at    | datetime     | 操作时间     |          |

# 三、核心业务关系设计

本系统采用“用户中心 + 企业招聘 + 简历投递 + 面试沟通 + AI匹配”整体架构设计。

系统核心业务关系如下：

---

# 3.1 用户（users）与企业（companies）

## 关系说明

系统中的 HR 用户可以绑定企业信息。

一个企业可以拥有多个 HR 用户。

## 关系类型

```text
companies (1) ------ (N) users
```

## 业务场景

- 企业注册后创建企业主页
- HR加入企业后可发布职位
- 企业统一管理多个招聘人员

## 关键关联字段

| 表        | 字段       | 说明       |
| --------- | ---------- | ---------- |
| users     | company_id | 所属企业ID |
| companies | id         | 企业主键   |

---

# 3.2 企业（companies）与职位（jobs）

## 关系说明

一个企业可以发布多个招聘职位。

职位属于具体企业。

## 关系类型

```text
companies (1) ------ (N) jobs
```

## 业务场景

- 企业发布招聘岗位
- 企业管理职位状态
- 展示企业职位列表

## 关键关联字段

| 表        | 字段       | 说明     |
| --------- | ---------- | -------- |
| jobs      | company_id | 企业ID   |
| companies | id         | 企业主键 |

---

# 3.3 用户（users）与简历（resumes）

## 关系说明

求职者可以创建多份简历。

支持不同岗位投递不同简历。

## 关系类型

```text
users (1) ------ (N) resumes
```

## 业务场景

- 校招简历
- 社招简历
- 不同行业简历模板

## 关键关联字段

| 表      | 字段    | 说明     |
| ------- | ------- | -------- |
| resumes | user_id | 求职者ID |
| users   | id      | 用户主键 |

---

# 3.4 简历主表与子表关系

## 关系说明

简历采用“主表 + 子表”结构设计。

一个简历可以包含：

- 多条教育经历
- 多条工作经历
- 多个项目经历

## 关系类型

```text
resumes (1) ------ (N) resume_educations
resumes (1) ------ (N) resume_experiences
resumes (1) ------ (N) resume_projects
```

## 设计优势

- 避免字段冗余
- 提高扩展性
- 适合复杂简历结构

---

# 3.5 职位（jobs）与投递记录（applications）

## 关系说明

求职者通过简历向职位投递申请。

一个职位会收到多个投递记录。

一个用户也可以投递多个职位。

因此：

用户与职位属于“多对多关系”，通过 applications 中间表维护。

## 关系类型

```text
users (N)
           \
            >------ applications ------< jobs (N)
           /
resumes (N)
```

## 业务场景

- 用户投递职位
- HR查看候选人
- 招聘流程流转
- 录用管理

## 关键关联字段

| 表           | 字段       | 说明     |
| ------------ | ---------- | -------- |
| applications | user_id    | 求职者ID |
| applications | job_id     | 职位ID   |
| applications | resume_id  | 简历ID   |
| applications | company_id | 企业ID   |

---

# 3.6 投递记录（applications）与面试邀约（interview）

## 关系说明

一次投递可以产生多轮面试。

例如：

- 一面
- 二面
- HR终面

因此：

applications 与 interview 属于一对多关系。

## 关系类型

```text
applications (1) ------ (N) interview
```

## 业务场景

- 面试通知
- 面试确认
- 面试状态流转
- 多轮面试管理

## 关键关联字段

| 表           | 字段           | 说明       |
| ------------ | -------------- | ---------- |
| interview    | application_id | 投递记录ID |
| applications | id             | 投递主键   |

---

# 3.7 用户聊天关系（conversations 与 messages）

## 关系说明

系统支持 HR 与求职者即时沟通。

会话表保存聊天会话。

消息表保存聊天消息。

## 关系类型

```text
conversations (1) ------ (N) messages
```

## 业务场景

- 在线沟通
- 面试通知
- 招聘消息提醒
- WebSocket实时聊天

## 关键关联字段

| 表            | 字段            | 说明     |
| ------------- | --------------- | -------- |
| messages      | conversation_id | 会话ID   |
| conversations | id              | 会话主键 |

---

# 3.8 企业人才库关系（company_resume）

## 关系说明

企业可以收藏求职者进入人才库。

属于企业与求职者的多对多关系。

## 关系类型

```text
companies (N)
             \
              >------ company_resume ------< users (N)
             /
         resumes
```

## 业务场景

- 收藏优秀候选人
- 建立企业人才池
- 后续岗位推荐

---

# 3.9 用户收藏职位关系（job_favorites）

## 关系说明

用户可以收藏多个职位。

职位也会被多个用户收藏。

属于典型多对多关系。

## 关系类型

```text
users (N)
           \
            >------ job_favorites ------< jobs (N)
           /
```

## 业务场景

- 收藏岗位
- 稍后投递
- 岗位跟踪

---

# 3.10 AI智能匹配关系（match_records）

## 关系说明

AI模块会自动分析：

- 岗位技能要求
- 简历技能
- 工作经验
- 学历
- 薪资匹配度

并生成匹配评分。

## 关系类型

```text
jobs (N)
         \
          >------ match_records ------< resumes (N)
         /
```

## 技能匹配明细关系

```text
match_records (1) ------ (N) match_skill_details
```

## 业务场景

- AI推荐人才
- AI推荐职位
- 智能排序
- 人岗匹配分析

---

# 3.11 系统反馈关系（feedbacks）

## 关系说明

用户可以提交：

- Bug反馈
- 功能建议
- 举报信息

反馈记录与用户关联。

## 关系类型

```text
users (1) ------ (N) feedbacks
```

---

# 3.12 系统日志关系（operation_logs）

## 关系说明

系统记录所有后台关键操作。

用于：

- 安全审计
- 行为追踪
- 管理员操作记录

## 关系类型

```text
users (1) ------ (N) operation_logs
```

---

# 3.13 整体ER核心结构（简化版）
